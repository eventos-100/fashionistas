## Quick Setup Commands

```bash
# 1. Clone and navigate to example
git clone --depth 1 --filter=blob:none --sparse https://github.com/CopilotKit/CopilotKit.git
cd CopilotKit
git sparse-checkout set examples/copilot-form-filling
cd examples/copilot-form-filling

# 2. Install dependencies
pnpm install

# 3. Configure environment
echo "NEXT_PUBLIC_COPILOT_PUBLIC_API_KEY=your_api_key_here" > .env

# 4. Run development server
pnpm dev
```

## Core Implementation

### 1. CopilotKit Provider Setup
Location: `app/layout.tsx`

```tsx
<CopilotKit 
  publicApiKey={process.env.NEXT_PUBLIC_COPILOT_PUBLIC_API_KEY}
  showDevConsole={false}
>
  {children}
</CopilotKit>
```

### 2. Form State Sharing with AI
Using `useCopilotReadable` to share form state:

```tsx
useCopilotReadable({
  description: "The security incident report form fields and their current values",
  value: form
}, [form]);
```

### 3. AI Form Actions
Using `useCopilotAction` to enable AI form updates:

```tsx
useCopilotAction({
  name: "fillIncidentReportForm",
  description: "Fill out the incident report form",
  parameters: [
    // Define form field parameters
  ],
  handler: async (action) => {
    // Update form values
  }
});
```

## Fashion Directory Integration

### Form Types for Fashion Directory
1. **Designer Profile Form**
   - Name, bio, specialties
   - Contact information
   - Portfolio links
   - Awards and achievements

2. **Event Registration Form**
   - Event details
   - Venue information
   - Schedule
   - Participant limits

3. **Fashion Show Application**
   - Designer information
   - Collection details
   - Technical requirements
   - Media assets

### Implementation Steps

1. **Create Fashion-Specific Forms**
```tsx
// Example: Designer Profile Form
const designerFormSchema = z.object({
  name: z.string().min(2),
  bio: z.string().min(50),
  specialties: z.array(z.string()),
  portfolio: z.string().url(),
  experience: z.number(),
  // Add more fields
});
```

2. **Configure AI Context**
```tsx
useCopilotReadable({
  description: "Fashion designer profile form with industry-specific fields",
  value: {
    formData: form.watch(),
    industryContext: "fashion design",
    formType: "designer_profile"
  }
});
```

3. **Customize AI Actions**
```tsx
useCopilotAction({
  name: "fillDesignerProfile",
  description: "Fill designer profile with fashion industry context",
  parameters: [
    {
      name: "designerName",
      type: "string",
      description: "Full name or brand name"
    },
    // Fashion-specific parameters
  ],
  handler: async (action) => {
    // Update form with fashion-specific logic
  }
});
```

### Best Practices for Fashion Forms
1. **Industry Terminology**: Train AI with fashion-specific terms
2. **Visual Elements**: Consider image upload fields for portfolios
3. **Validation Rules**: Industry-specific validation (e.g., valid fashion weeks)
4. **Multi-language**: Support for international fashion community
5. **Accessibility**: Ensure forms work with screen readers
