# Contributing Guidelines

## Code Style
- Use TypeScript strict mode
- Follow ESLint rules
- Use Prettier for formatting

## Commit Messages
Follow conventional commits:
- feat: New feature
- fix: Bug fix
- docs: Documentation
- style: Formatting
- refactor: Code restructuring
- test: Tests
- chore: Maintenance

## Pull Requests
1. Fork the repository
2. Create feature branch
3. Make changes
4. Run tests
5. Submit PR
